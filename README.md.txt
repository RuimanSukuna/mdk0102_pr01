# Практическая работа №1 — Анализ требований к приложению

**Вариант:** №15  
**Тема задания:** Панель администратора для управления пользователями  
**Студент:** 30ip235

---

##  Задание
Разработать панель администратора для управления пользователями системы.

---

##  Анализ требований

### Бизнес-требования
- Повысить эффективность управления пользователями
- Снизить время на ручные операции с учётными записями
- Обеспечить безопасность и контроль доступа к данным пользователей

### Функциональные требования
- Просмотр списка пользователей с фильтрацией и поиском
- Добавление новых пользователей
- Редактирование данных пользователей
- Блокировка/разблокировка учётных записей
- Сброс паролей пользователям
- Назначение ролей и прав доступа
- Экспорт списка пользователей (CSV/Excel)
- Просмотр истории действий пользователя

### Нефункциональные требования
- Отклик интерфейса ≤ 2 секунд
- Поддержка 100+ одновременных администраторов
- Данные шифруются при передаче (HTTPS)
- Резервное копирование данных ежедневно
- Совместимость с Chrome, Firefox, Safari последних версий
- Логирование всех действий администратора

---

##  User Story
**US1:** Как администратор, я хочу фильтровать пользователей по статусу, чтобы быстро находить заблокированные учётные записи.

**US2:** Как администратор, я хочу массово назначать роли пользователям, чтобы экономить время при onboarding.

**US3:** Как администратор, я хочу видеть историю входов пользователя, чтобы выявлять подозрительную активность.

---

##  Acceptance Criteria

**Для US1:**
Given: Я нахожусь в разделе "Пользователи"
When: Выбираю фильтр "Статус: Заблокирован"
Then: В списке отображаются только заблокированные пользователи


**Для US2:**
Given: Выбрано несколько пользователей
When: Нажимаю "Назначить роль" и выбираю "Модератор"
Then: Все выбранные пользователи получают роль "Модератор"
And: Система показывает уведомление об успешном выполнении


**Для US3:**
Given: Я открываю карточку пользователя
When: Перехожу во вкладку "История входов"
Then: Вижу таблицу с датой, временем и IP-адресом последних 10 входов



##  Возможные неясности / риски

1. **Неясность:** Какие роли по умолчанию должны быть в системе?  
   **Риск:** Без чёткого списка ролей невозможно настроить права доступа.

2. **Неясность:** Что именно можно редактировать в профиле пользователя?  
   **Риск:** Администратор может изменить критичные данные (email без подтверждения).

3. **Неясность:** Нужна ли мягкая блокировка (временная) или только полная?  
   **Риск:** Пользователи могут быть заблокированы навсегда из-за ошибки.

4. **Неясность:** Какие данные должны попадать в экспорт?  
   **Риск:** Может быть экспортирована конфиденциальная информация.
